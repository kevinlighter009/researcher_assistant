# DiLu: A Knowledge-Driven Approach to Autonomous Driving with Large Language Models

## Keywords
- llm-agent, knowledge-driven, memory-module, reflection, chain-of-thought, closed-loop-decision, highway-env, few-shot

## TL;DR
Data-driven autonomous driving stacks suffer from dataset bias, overfitting, and uninterpretability, and RL-based decision policies fail to generalize across environments. The authors propose DiLu, an LLM-based driver agent that combines a Reasoning module (Chain-of-Thought + few-shot retrieval from a Memory module), a Reflection module (LLM self-correction of unsafe decisions), and a vector-keyed Memory module that continuously accumulates driving experiences. With only 40 memory items, DiLu matches or beats GRAD (a SOTA RL baseline trained for 600,000 episodes) on Highway-env, and exhibits much stronger transfer to denser-traffic / different-lane settings (35% SR vs 10% SR for GRAD on lane-5-density-3).

## Problem & Motivation
Recent autonomous driving relies on data-driven methods that "always encounter challenges such as dataset bias, overfitting, and uninterpretability" (Codevilla 2019, Jin 2023). RL-based decision-makers in particular tend to overfit the specific environment they are trained on and fail to generalize to denser traffic or extra lanes. Inspired by LeCun (2022) — "Why can an adolescent learn to drive a car in about 20 hours of practice and know how to act in many situations she has never encountered before?" — the authors argue that human driving is fundamentally knowledge-driven, not data-driven. The paper therefore asks how to instill knowledge-driven capability into autonomous driving by leveraging LLMs' common-sense knowledge, in-context learning, and CoT reasoning.

## Innovation Points
- **Knowledge-driven paradigm for AD** — formalizes a closed loop of *Environment + Driver Agent (Recall / Reason / Reflect) + Memory* as the structural alternative to data-driven AD.
- **Reasoning module** — Scenario Descriptor turns Highway-env state into natural-language scene text; a Prompt Generator combines system prompt + scene description + top-k recalled few-shot experiences; LLM produces a CoT response; Decision Decoder maps to a discrete Highway-env action.
- **Reflection module** — after each driving session, key safe frames are summarized and stored, while unsafe (collision-causing) frames are passed back through the LLM as a *mistake-rectifier* to produce revised reasoning + revised decision, which then enrich Memory. The memory thus contains both `success_memory` and `correction_memory`.
- **Vector-keyed Memory module** — each memory item is a (scene-description embedding, reasoning process) pair; recall is by similarity search (Johnson 2019, FAISS-style) of the current scene embedding.
- **Cross-domain experience transfer** — because memory is stored as natural-language text rather than weights, experiences extracted from CitySim real-world trajectory data can be directly loaded into the Memory module and used in Highway-env.

## Model Architecture
- Environment: Highway-env motorway simulator (configurable lanes and vehicle density), discrete actions (e.g. IDLE, accelerate, lane-change-left/right).
- Reasoning module:
  - Scenario Descriptor: state (ego speed, lane id, surrounding vehicles' speed/acceleration/lane position) → templated natural-language scene description.
  - Memory recall: scene description embedded into a vector; top-k (few-shot) similar past (scene, reasoning) pairs retrieved from Memory.
  - Prompt Generator: system prompt + textual scene description + few-shot experiences.
  - LLM (out-of-the-box, prompted with Chain-of-Thought) produces step-by-step reasoning ending in a final action decision.
  - Decision Decoder: parses LLM output into a discrete action and feeds it to the environment, repeating frame-by-frame to form a closed loop.
- Reflection module:
  - On a successful session, Key-Frame Sampling picks several safe decision frames and stores them as `success_memory`.
  - On an unsafe session (collision), a Mistake Correction step prompts the LLM with the unsafe frame + original reasoning, asks it to identify the mistake and propose corrected reasoning + decision, and stores the revised frame as `correction_memory`.
- Memory module: vector store keyed by scene-description embedding; values are natural-language reasoning traces; initialized with a few human-crafted examples (akin to "driving school"), then grown by the Reflection module.
- Inputs: symbolic Highway-env state (no raw camera). Outputs: discrete high-level driving action per frame.
- LLM size / which model: not reported in the main paper text (the paper says "out-of-the-box LLM"); decision-making latency reported as 5–10 s per frame.

## Benchmark Results
Environment: Highway-env (Leurent 2018). Primary metrics: Success Steps (SS) — number of consecutive collision-free decision frames out of 30 (so SS = 30 means session completed); Success Rate (SR) — fraction of 10 test scenarios reaching 30 collision-free decision frames.

**Effect of memory size and few-shot count (lane-4-density-2; box plots over 10 seeds):**
- 0-shot baseline (no Memory): no tasks succeed, median SS below 5 — confirms LLM cannot drive zero-shot.
- 5-shot setting: completes the task in the majority of cases with 20 experiences in memory; with 40 experiences, all trials achieve median SS ≥ 25.
- 3-shot / 1-shot at 40 memory items: median SS = 27 / 25 respectively.

**Comparison with RL baseline GRAD (Xi & Sukthankar 2022), trained on lane-4-density-2 for 600,000 episodes:**
| Setting              | DiLu SR | GRAD SR |
|----------------------|---------|---------|
| lane-4-density-2 (in-domain) | 70%     | 69%     |
| lane-5-density-2.5 (transfer) | 65%     | 47%     |
| lane-5-density-3 (transfer)   | **35%** | 10%     |

DiLu uses only 40 experiences (vs 600,000 RL episodes). RL failures attributed mainly to inability to brake in time.

**Generalization (20 memory items from lane-4-density-2, evaluated on lane-5-density-3):**
- 3-shot: median SS drops from 13 (in-domain) to 5 (transfer).
- 5-shot: ~23 median SS in transfer setting → more few-shot examples improves generalization.

**Real-world transfer (CitySim trajectory data, 20 experiences extracted):**
- Memory built from CitySim outperforms a memory built only from Highway-env when transferring to lane-5-density-3, beating the no-experience baseline (Figure 7b).

**Reflection ablation (Table 1, both memory types vs. baseline 20-experience module):**
| Methods               | MIN | Q1   | Median | Q3   | MAX |
|-----------------------|-----|------|--------|------|-----|
| baseline              | 2.0 | 3.0  | 10.0   | 22.0 | 30.0 |
| +success_memory       | 3.0 | 7.0  | 24.5   | 29.3 | 30.0 |
| +correction_memory    | 4.0 | 7.75 | 21.5   | 30.0 | 30.0 |
| +both_type_memory     | 5.0 | 7.8  | 24.5   | 30.0 | 30.0 |

Both reflection-derived memory types improve over baseline; combining them is best.

## Limitations & Open Questions
- Decision-making latency 5–10 s per frame (LLM inference + API), incompatible with control-rate driving — explicitly acknowledged.
- The framework "does not completely eradicate hallucinations generated by LLMs" — explicitly acknowledged.
- Evaluated only in Highway-env (a symbolic-state simulator) plus CitySim trajectories; no perception stack, no real sensors, no real-world closed-loop test.
- Action space is discrete high-level meta-actions; coupling to a low-level controller / continuous control is out of scope.
- Specific LLM identity, parameter count, and inference cost not reported in the main paper; reproducibility on smaller open LLMs not characterized.
- Memory grows without an explicit forgetting / deduplication policy described in the main text.
